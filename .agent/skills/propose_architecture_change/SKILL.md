---

Use quando o trabalho envolver reorganizacao estrutural, novos componentes ou mudancas de contrato do projeto.

Execute:

1. Ler o contexto canonico do projeto e os artefatos impactados
2. Explicar o problema arquitetural e o impacto esperado
3. Propor a menor mudanca coerente com o modelo canonico da Cindy
4. Solicitar aprovacao explicita antes de editar qualquer arquivo

Regras:

- Considerar esta skill como risco medio
- Nao implementar antes da aprovacao do PO
- Nao importar estruturas especificas de outros projetos
