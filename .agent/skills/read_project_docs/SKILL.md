---
name: read_project_docs
description: Ler e analisar a documentacao canonica e os artefatos do projeto antes de executar qualquer trabalho.
---

Use quando o objetivo for entender contexto, escopo, regras, sprint ativa ou evidencias do projeto.

Execute:

1. Ler `README.md`, `rules/WORKSPACE_RULES.md` e regras globais aplicaveis
2. Ler `Dev_Tracking.md` e o `Dev_Tracking_SX.md` ativo, se existir
3. Ler apenas os docs canonicos necessarios em `docs/`
4. Consultar `tests/bugs_log.md` quando houver impacto em bugs, testes ou rastreabilidade
5. Resumir contexto, limites e proximo passo antes de editar

Regras:

- Priorizar caminhos canonicos do projeto
- Manter leitura proporcional ao escopo
- Nao editar arquivos durante esta skill
