# Workflows Comuns de Jira

## 1. Investigacao Read-Only

Use quando o objetivo for entender a situacao sem alterar nada.

Checklist:

- confirmar projeto, board ou issue-alvo
- buscar somente os itens relevantes
- resumir status atual, bloqueios, responsavel e proximo passo sugerido
- nao comentar, nao transicionar e nao editar campos

Saidas uteis:

- quadro resumido por status
- issues prioritarias
- bloqueios e dependencias
- gaps de especificacao

## 2. Triagem de Backlog

Use quando o objetivo for organizar trabalho pendente.

Foco:

- prioridade
- clareza da descricao
- dependencias
- falta de assignee
- labels ausentes
- risco de issue parada ou ambigua

Entregue:

- itens que precisam de clarificacao
- itens que precisam de prioridade
- itens bloqueados ou sem dono
- recomendacoes sem executar alteracoes

## 3. Acompanhamento de Sprint

Use para:

- listar em andamento
- identificar bloqueadas
- encontrar issues sem atualizacao
- resumir progresso por status ou assignee

Se houver pedido read-only, apenas reportar.

## 4. Atualizacao Operacional

Antes de qualquer alteracao:

- citar issue-alvo
- citar campos afetados
- citar o valor atual e o valor pretendido, se conhecido
- confirmar se ha aprovacao explicita do usuario

## 5. Criacao ou Refinamento de Issue

Use quando o usuario pedir para abrir ou melhorar um item.

Checklist:

- tipo de issue
- titulo claro
- descricao objetiva
- criterios ou escopo
- labels, prioridade e assignee se conhecidos
- dependencias ou contexto de negocio

## 6. Relatorio Operacional

Use para resumir backlog, sprint, release ou fila de atendimento.

Foco:

- volume por status
- itens criticos
- gargalos
- risco de prazo
- proximas acoes sugeridas

## 7. Estrutura de Resumo Recomendado

- contexto
- status atual
- bloqueios ou dependencias
- risco
- proxima acao sugerida
