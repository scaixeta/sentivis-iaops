Fluxo:

1. Ler contexto e templates
2. Planejar mudanças por documento
3. Aguardar aprovação explícita
4. Editar docs canônicos
5. Validar ordem e conformidade DOC2.5

Restrições:

- Não criar `docs/README.md`
- Não criar `docs/INDEX.md`
- Manter ordem de ciclo de vida dos 4 docs
