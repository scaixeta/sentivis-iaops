- Inicialização de contexto: usar `doc25-init`
- Desenvolvimento: usar `doc25-dev-workflow`
- Documentação: usar `doc25-docs-workflow`
- Commit/push: usar `doc25-commit-gate`

Aplicar sempre:

1. Entendimento
2. Planejamento
3. Aprovação explícita
4. Execução
5. Rastreabilidade
