# OPERATIONS

## Proposito

Orientar como operar, validar e manter o projeto.

## Visao geral de operacao

- ambiente principal de execucao
- artefatos que precisam permanecer saudaveis
- rotina minima de validacao

## Rotinas de teste

### Testes manuais minimos

- validar `README.md`
- validar `docs/`
- validar tracking e `bugs_log`

### Testes automatizados

- documentar o comando real do projeto
- nao citar testes inexistentes

## Rotinas operacionais

- validar estrutura DOC2.5
- validar espelho de skills, se existir
- validar consistencia de workflows e regras
- registrar falhas em `tests/bugs_log.md`

## Seguranca operacional

- nunca versionar credenciais
- nunca documentar segredos
- mascarar valores sensiveis

## Resposta a falhas

1. confirmar contexto da sprint
2. registrar bug ou teste
3. corrigir o artefato minimo necessario
4. atualizar `Timestamp UTC`
