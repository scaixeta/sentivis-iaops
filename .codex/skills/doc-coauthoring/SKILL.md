# Doc Coauthoring

## Quando usar
Usar para criar/revisar documentação canônica de projeto e meta-projeto com linguagem técnica objetiva.

## Procedimento
1) Confirmar documento canônico alvo (`SETUP/ARCHITECTURE/DEVELOPMENT/OPERATIONS`).
2) Consolidar conteúdo sem duplicar arquivos.
3) Preservar histórico de sprint e rastreabilidade.
4) Validar consistência com regras globais em `.clinerules/`.

## Restrições DOC2.5
- Não criar `docs/README.md` em projetos DOC2.5.
- Evitar tabelas complexas e formatos fora do padrão.
- Commit só por comando do PO.

## Dependências
- Documentos canônicos do projeto
- Regras de governança DOC2.5
